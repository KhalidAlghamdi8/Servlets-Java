In this example, filters are used to add data to the request for providing the information required of the header and footer in the page.
This is particularly useful since this data is presented in every page that uses the template files. Instead of using parameters or having a servlet, this information is placed on the request. You may use filters to include the data that is displayed in the pages.
This page displays values stored in the request that were set by the filter.
Since the attributes set has a - sign you cannot use the . syntax.
Like in the previous example, if there's an attribute that contains an Expression Language operator in the name (or spaces), you must use the ['NAME OF THE ATTRIBUTE'] syntax.
