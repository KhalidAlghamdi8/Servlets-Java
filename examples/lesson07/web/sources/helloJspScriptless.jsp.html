<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
<%@page contentType="text/html" pageEncoding="UTF-8" %>
<!DOCTYPE html>
<html>
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    <link rel="stylesheet" href="res/styles.css" type="text/css"/>
    <title>JSP Page</title>
  </head>
  <body>
    <c:set var="name" value="world" />
    <c:if test="${not empty param.name}">
      <c:set var="name" value="${param.name}" />
    </c:if>
    <h1>Hello ${name}!</h1>
    <a href="index.html">Go home</a>
  </body>
</html>
One of the most important advantages of scriptless JSP is that document formatting is easier to read. Formatting the code that is generated results in clearer indentation and better organization of the elements inside.
Since there is no Java code mixed inside, tag alignment inside the Java code is the only worry.
In scriptless JSP, using Expression Language and custom tags (instead of Java code) allows you to perform some of the most common tasks where Java would be normally used.
To use a custom JSP tag, the tag library must be added to the JSP page. Think of this like "an import" for tags.
The prefix defines the way tags from this library must be used. In this case, all the custom tags from this library must start with <c:
The URI is the taglib identifier. This is a unique identifier for tag libraries. In this case, the core Java Standard Tag Library URI is: http://java.sun.com/jsp/jstl/core
Page directives are used the same way as with other JSP pages.
In the servlet and the scriptful JSP, there is a constant declared. Constants do not exist in scriptless JSP. In this case, the constant value is assigned to the variable name.
Instead of adding Java scripting for if evaluation, the c:if tag may be used. Contents of this tag are executed or displayed if the evaluation test attribute of this is true.
Inside the test attribute of this tag, there is Expression Language code. Contents inside ${ } is evaluated (as with the <%= script expressions)
The evaluation for this if statement is true if the request parameter name is not empty (not an empty string or null).
If sending the request parameter 'name', then the name variable for this JSP page is assigned.
Expression Language may be used outside custom tags. When done this way, the results of the expression are part of the normal output, like the <%= Script expression
However, Expression Language is a "smarter" choice for handling expression. Take null value handling as an example. If there is a null value at some point in the expression, an empty string ('') is written to the output instead of writing 'null' or throwing an exception .
