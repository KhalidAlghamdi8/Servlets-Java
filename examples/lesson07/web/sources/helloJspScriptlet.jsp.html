<%@page contentType="text/html" pageEncoding="UTF-8"%>
<!DOCTYPE html>
<%! private static final String DEFAULT_NAME = "world";%>
<html>
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    <link rel="stylesheet" href="res/styles.css" type="text/css"/>
    <title>helloJspScriptlet</title>
  </head>
  <body>
    <%
      String name = request.getParameter("name");
      if (name == null) {
        name = DEFAULT_NAME;
      }
      %>
    <h1>Hello <%=name%>!</h1>
    <a href="index.html">Go home</a>
  </body>
</html>
HTML in JSP is written "as is." In JSP, everything that is not inside the special "script" tags is output "as is."
In JSP constants, instance variables and methods may be declared inside <%! %> tags.
This is a JSP java scriptlet. Valid Java code inside <% %> is added to the service handling method of the JSP. Java code inside this tags is inserted in the resulting class where the tag is.
It is possible to add for loops, if conditions, method calls, and anything that is usually placed inside the doGet or doPost methods of a servlet.
This is a JSP Java expression. Java code inside <%= %> must evaluate to a value (if given an object, the toString() method is used)
This is a JSP Page Directive. Page Directives define additional data that the page needs. In this case, the contentType and encoding headers are set using the directive.
Another use is to add imports and custom tag libraries. Directives are usually placed at the top of a JSP file.
