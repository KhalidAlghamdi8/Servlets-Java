Let's review this project in NetBeans.

NetBeans WebProject Structure.

Take some time to review what the structure of a web application in NetBeans IDE.

In the Projects tab you can easily explore the contents of your application. This is how an application looks like in NetBeans.
Hover over the elements to see a description.

  • WebProject
    This is the project root. All resources, configuration files, and classes are part of the project.
    • Web Pages
      This folder contains all the web resources in your application.
      A resource may be:
      • HTML pages
      • JavaScript files
      • Images
      • JSP files.
      • WEB-INF
        The WEB-INF folder contains the configuration files for the web application as well as the libraries and compiled classes.
        This folder is not accessible, in fact trying to access it will return a simple 404 error in your browser as it wont be a direct part of the web application.
        Although its not served directly the contents of this folder are processed by the server, classes inside are loaded and libraries are included as part of the classpath.
        • lib
          In addition of using NetBeans for library management you can manually add libraries to your application by adding .jar files in the WEB-INF/lib directory.
          • Libraries as .Jar files.
        • beans.xml
          This file contains the Contexts and Dependency Injection configuration.
        • web.xml
          The web.xml file its the most important configuration file for a Web Application. If annotations are not used for configuration this file needs to contain all the information for the Web Application configuration.
        • weblogic.xml
          The weblogic.xml file defines specific parameters that Oracle's WebLogic Server uses for deployment of the application.
        • glassfish.xml
          Other server specific XML files can be added to the WEB-INF folder.
        • OTHER FILES
          Other configuration files are usually located inside the WEB-INF folder. You can also include resources that cannot be accessed directly by URL
          For example, you may add images or JSP files here and access them using Java Servlets.
      • HTML pages
      • JSP pages
      • Javascript files
      • Images
      • More folders for organizing content
    • Source Packages
      Java Packages and classes are stored inside this folder.
      This folder is organized by packages to make easier the modification of Java classes.
      • java.package...
        • JavaClasses
    • Libraries
      If handled by NetBeans, libraries and dependencies are organized in this folder.
    • RESTful Web Services
      REST Services declared in the application are organized here.
    • Configuration Files
      Configuration files are conveniently organized in this folder (they are the same as the original)
      • MANIFEST.MF
        The MANIFEST file for the Application Bundle.
      • beans.xml
      • web.xml
      • weblogic.xml
