Application initialization parameters are set in the web.xml file.
Multiple parameters are supported and can be added one after another.
It's a good idea to keep all the context parameters together to keep the file easy to read.
When you want to set up a servlet using xml you have to declare it using the servlet element. The name is very important since it is used in the later servlet-mapping element to map it to an URL.
You may set initialization parameters for your servlet by adding multiple init-param elements.
The servlet mapping is the same as the WebServlet.urlPattern annotation counterpart.
The main difference is that when declared in the web.xml file, this element is completely independent from the servlet declaration in the servlet element.
It is important to match the servlet-name in the servlet element with the one in the servlet-mapping element.
To map a servlet to multiple resources, add multiple servlet-mapping elements for the same servlet.
You can use wildcards to map resources as well. Example: *.do (for a servlet that handles forms)
