<!DOCTYPE html>
<html>
  <head>
    <title>Hello</title>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    <link rel="stylesheet" href="res/styles.css" type="text/css"/>
  </head>
  <body>
    <h1>Hello!</h1>
    <form action="HelloSvlt" method="POST">
      <table>
        <tr>
          <td>What's your name? </td>
          <td><input type="text" name="username"/></td>
        </tr>
        <tr>
          <td>What's your role:</td>
          <td>
            <select name="role">
              <option value="student">Student</option>
              <option value="instructor">Instructor</option>
              <option value="adm">Administrator</option>
            </select>
          </td>
        </tr>
        <tr>
          <td colspan="2">
            <input type="submit" value="Submit">
          </td>
        </tr>
      </table>
    </form>
    <p>
      <a href="index.html">Go back</a>
    </p>
  </body>
</html>
This is a standard HTML form. To use it with servlets, the action must be the URL that the servlet is mapped to.
The Java method specifies the HTTP method used for the call and maps to the doMETHOD.
  • method="GET" in the HTML form invokes the doGet method in the servlet.
  • method="POST" in the HTML form invokes the doPost method method in the servlet.
Fields are sent as parameters to the servlet by name. In this case, the form field is sent as the request parameter "name"
The "role" field is sent to the servlet in the same manner the "name" field is.
Forms are usually submitted using a Submit input field. This field is rendered as a button in the browser.
