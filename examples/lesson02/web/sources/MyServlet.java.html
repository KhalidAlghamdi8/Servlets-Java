package com.examples.lesson02;

import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet(name = "MyServlet", urlPatterns = {"/MyServlet"})
public class MyServlet extends HttpServlet {

  @Override
  protected void doGet(HttpServletRequest request, HttpServletResponse response)
          throws ServletException, IOException {
    request.setAttribute("myText", "Hello World!");
    request.getRequestDispatcher("/myJsp.jsp").forward(request, response);
  }
}
To configure a servlet, use the @WebServlet annotation. It defines the name of the servlet and the URLs the servlet is mapped to.
A servlet is a class that extends the HttpServlet class and implements some methods to handle requests.
The doGet method handle requests that are made using the browser's address bar.
In this implementation the servlet stores a message that is displayed by the myJsp.jsp page using: 
request.setAttribute("myText", "Hello World!");
And then delegates the html page layout and code to the jsp using: 
request.getRequestDispatcher("/myJsp.jsp").forward(request, response);
Servlets are handle the logic behind dynamic web pages. They invoke services that in turn perform business logic and prepare the results of this business operations to display them in a page.
