<fmt:setLocale value="${param.lang}"/>
<fmt:setBundle basename="com.examples.lesson08.i18n.shopList" var="messages"/>

<h4><fmt:message bundle="${messages}" key="madeWith"/></h4>

<fmt:message var="title" bundle="${messages}" key="title"/>
<jsp:include page="templateHeader.jsp">
  <jsp:param name="title" value="${title}"/>
</jsp:include>

  <thead>
    <tr>
      <th><fmt:message bundle="${messages}" key="item"/></th>

The FMT library contains tags that aid in formatting data and using localization in JSP files.
The setLocale tag sets the language for the messages.
In Java, localization is commonly implemented with properties file.
These files are stored inside packages. The file name for the properties file contain the language code as a suffix.
The setBundle tag specified in the path is the basename attribute, just like in a Java class.
In this case, the properties file that is loaded is /com/examples/lesson08/i18n/shopList_LANG.properties, where LANG is replaced with the language code set by the setLocale tag.
The resource bundle is loaded in a variable available in the page using Expression Language.
The message tag prints the message from the bundle variable. Note that you can not use only Expression Language to display messages from a bundle variable. This tag displays the title message.
This tag displays the Item table header.
This tag displays the Quantity table header.
This is a message that uses a parameter. Parameters are frequently used for complex messages. In this case, the message expects the number of items in the shopping list.
Parameters are sent in order and they are used in bundle files by index using {#} braces.
Message tags may be used inside other tags. In this case, the page imports the footer page and sends a parameter that comes from the message bundle.
See that the message also receives a parameter.
The message expects a date, but since dates may be also localized, the date must also be formatted.
Note that formatting of raw values is possible with FMT tags as well.
