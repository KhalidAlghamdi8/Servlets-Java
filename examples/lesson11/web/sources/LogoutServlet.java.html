package com.examples.lesson11.servlets;

import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet(name = "LogoutServlet", urlPatterns = {"/LogoutServlet"})
public class LogoutServlet extends HttpServlet {

  @Override
  protected void doGet(HttpServletRequest request, HttpServletResponse response)
          throws ServletException, IOException {
    try {
      request.logout();
      request.getSession().invalidate();
    } catch (Exception e) {
    }
    response.sendRedirect(getServletContext().getContextPath() + "/index.jsp");
  }
}
There are two options for processing a logout,
  • request.logout() removes only the user credentials and roles from the user, leaving all data in the session available.
    Use this approach when you want to have data available to the user after logging out of and the data stored in the session is user-independent.
  • session.invalidate() destroys the user's session and removes all data (including login credentials) from the session.
    Use this approach if user-dependent information was stored in the session.
This example shows both approaches, although only one of them is necessary.
You can optionally perform a request.logout() and then remove the user-dependent information from the session manually.
