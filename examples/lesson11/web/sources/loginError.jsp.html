The login error page may contain anything from a simple page to a complex page that displays data.
In this case, the login form is displayed again that allows a user to retry. An error messages is also shown. Note that although the login page is triggered by the server, authentication may be started at ANY time if you use the <form action="j_security_check" method="POST"> element.
After the authentication process is done, the user is redirected to the resource that requested the login (the page that submitted the login) or to the error page if the login fails.
