The header displayed in the example pages is generated by this JSP. It contains the navigation bars and does several validation on what content to display.
The request attributes this JSP needs are set by a filter that intercepts all the requests in the application, setting the current user and role as text in the request.
It uses a custom tag to mimic the request.isUserInRole method for displaying the tag contents if the user is in a particular role.
The RolesFilter that intercepts all the requests sets the userName and userRoles attributes.
The user is considered logged-in if the filter sets the username attribute. If it is not empty, the user is logged-in and then the user name, its roles and the logout link is displayed.
If the user is not logged-in, the main navigation bar with the different "sections" of the web application is displayed.
The sec:IfInRole tag is a custom tag (IfInRole.java) that processes the body of the tag if the user is in a particular role. In this case the, Quiz list is displayed if the user is a QuizUser
The sec:IfInRole tag is a custom tag (IfInRole.java) that processes the body of the tag if the user has a particular role. If the user is a quizAdmin, then the administration bar is displayed.
The sec:IfInRole tag is a custom tag (IfInRole.java) that processes the body of the tag if the user is in a particular role. If the user is a quizPublisher then the edit bar is displayed.
