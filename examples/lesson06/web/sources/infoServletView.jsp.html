<%@page contentType="text/html" pageEncoding="UTF-8"%>
<!DOCTYPE html>
<html>
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    <title>System information</title>
    <link rel="stylesheet" href="res/styles.css" type="text/css"> 
  </head>
  <body>
    <h1>System Information</h1>
    <p>JSP view</p>
    <ul>
      <li>Server Name: ${infoBean.serverName}</li>
      <li>Server Memory: ${infoBean.serverMemory}</li>
      <li>Server Free Memory ${infoBean.serverFreeMemory}</li>
      <li>Server Cores: ${infoBean.serverCores}</li>
      <li>Server Time: ${infoBean.serverTime}</li>
    </ul>
    <a href="index.html">Go Home</a>
  </body>
</html>
This is your first glance at a real JSP (with some dynamic code).
JSP pages are used to render HTML code and include dynamic data inside.
JSP are not meant to be used to make calculations or extensive processing, the correct approach is to do this processing in a servlet and then just using the JSP pages for rendering the elements as HTML.
It is easier to write HTML code in a JSP, but writing Java code in JSP is not advisable.
This is Expression Language and it is the easiest way to include attributes set in servlets inside a JSP page.
The value inside the ${ } is an expression. In this case the expressions refer to the request attribute infoBean (set by the Servlet).
Using Expression Language it is possible to call the get methods inside the infoBean object this is done using the dot operator in the expression. The expression: infoBean.serverTime will translate into the InfoBean.getServerTime method.
