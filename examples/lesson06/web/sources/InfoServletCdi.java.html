package com.examples.lesson06.servlets;

import com.examples.lesson06.model.InfoBean;
import java.io.IOException;
import java.io.PrintWriter;
import javax.inject.Inject;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet(name = "InfoServletCdi", urlPatterns = {"/infoCdi"})
public class InfoServletCdi extends HttpServlet {

  @Inject
  private InfoService infoService;

  @Override
  protected void doGet(HttpServletRequest request, HttpServletResponse response)
          throws ServletException, IOException {
    response.setContentType("text/html;charset=UTF-8");
    PrintWriter out = response.getWriter();
    out.println("<html>");
    out.println("<head>");
    out.println("<title>Servlet InfoServletCdi</title>");
    out.println("<link rel=\"stylesheet\" href=\"res/styles.css\" type=\"text/css\"/>");
    out.println("</head>");
    out.println("<body>");
    out.println("<h1>System information</h1>");
    out.println("<ul>");
    out.println("<li>Server Name: " + infoService.getServerName() + "</li>");
    out.println("<li>Server Memory: " + infoService.getServerMemory() + "</li>");
    out.println("<li>Server FreeMemory: " + infoService.getServerFreeMemory() + "</li>");
    out.println("<li>Server Cores: " + infoService.getServerCores() + "</li>");
    out.println("<li>Server Time: " + infoService.getServerTime() + "</li>");
    out.println("</ul>");
    out.println("<a href=\"index.html\">Go Home</a>");
    out.println("</body>");
    out.println("</html>");
  }
}
This servlet separates the business logic.
Instead of having the business logic here that logic is inside a Service Object that is injected into the Servlet.
Injection is the process of setting a variable with an Object created elsewhere.
In the case of Contexts and Dependency Injection (CDI), objects are created and managed by the server.
The goal of this class is to use that external object to get the system values.
The @Inject annotation allows the servlet to get an instance of the annotated variable.
In this case, this annotation tells the server that when this servlet is instantiated, it must set the infoService variable with a valid InfoService implementation.
It is a good practice to inject interfaces instead of concrete classes.
In this case, InfoServiceImpl is the concrete implementation of the InfoService interface.
The servlet gets the value from the InfoService. This way the servlet delegates all the business logic to the InfoService.
