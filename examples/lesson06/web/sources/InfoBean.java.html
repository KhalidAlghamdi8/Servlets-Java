package com.examples.lesson06.model;

public class InfoBean {

  private String serverMemory;
  private String serverFreeMemory;
  private String serverName;
  private String serverCores;
  private String serverTime;

  public String getServerMemory() {
    return serverMemory;
  }

  public void setServerMemory(String serverMemory) {
    this.serverMemory = serverMemory;
  }

  public String getServerFreeMemory() {
    return serverFreeMemory;
  }

  public void setServerFreeMemory(String serverFreeMemory) {
    this.serverFreeMemory = serverFreeMemory;
  }

  public String getServerName() {
    return serverName;
  }

  public void setServerName(String serverName) {
    this.serverName = serverName;
  }

  public String getServerCores() {
    return serverCores;
  }

  public void setServerCores(String serverCores) {
    this.serverCores = serverCores;
  }

  public String getServerTime() {
    return serverTime;
  }

  public void setServerTime(String serverTime) {
    this.serverTime = serverTime;
  }
}
This is the model bean. Model beans are sent to the view for display.
Model objects are generated by service classes, database queries or even the servlet. They are usually transfer objects that may be used to display some information.
Although not common, sometimes a model bean might contain basic validations or some business logic. Most of the time a model bean is a simple placeholder for values that might be persisted in a database or in some web scope.
